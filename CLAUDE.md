## GitHub Issues Protocol

Before starting ANY task, read `GITHUB_ISSUES_GUIDE.md` and follow it strictly.

- Use GitHub MCP tools (or `gh` CLI) to create a **master issue**, **phase issues**, and **granular sub-issues** for every project.
- Follow the **Hard Sync Rule** — after every completed step, check off the corresponding item in the GitHub sub-issue body, the phase issue body, and the local implementation plan `.md` file.
- Assign all issues to **Agoad1**.
- Apply relevant labels to every issue.
- Leave a completion comment on each sub-issue when it is closed, describing exactly what code was changed.
- Close sub-issues immediately upon completion; close phase issues when all their sub-issues are closed; close the master issue when all phases are complete.
